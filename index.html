    Introduction

    Welcome to the OpenCabs ‘Publishers API’

    This document describes the API to be used by 3rd parties (publishers) submitting bookings to the platform on behalf of their customers.

    The main difference with the “Customers APIs” is that the end customer is not required to register within the OpenCabs platform. It’s a user authorized by the publisher who is in charge of creating the booking for the end customer.

    Authentication

    OpenCabs uses API keys to allow access to the API. You can obtain an ApiKey by contacting us on support@opencabs.com.

    OpenCabs expects for the API key to be included in all API requests. You also need to provide a token, which will be used for authenticating your user profile, and the context (i.e. environment) for the execution of the request. This is required as the same user can be connected to several contexts (several publisher accounts for example).

    ApiKey: <your-api-key>
    Token: <your-token>
    ContextType”: "publisher"
    ContextId”: "<your-publisher-id>"

    OpenCabs supports JSON by default for the body of the requests / responses. You need to specify this in the headers with:

    Content-Type”: “application/json"

    You must replace your-api-key, your-api-key and your-publisher-id with your personal values.

    Bookings

    Create a new booking

    POST /book

    This API enables to submit new bookings to the platform.

    Generally you will need to make a pricing request to obtain a quoteId before submitting your booking. This is not required only if you are a platform and have been authorised to post jobs with a price set by yourself.

    Key fields

    Name Required Comments
    quoteId (see comments) The quoteId is required in all cases except if the publisher has been explicitly authorised (set up done by OpenCabs) for being able to post bookings without quoteId.
    In this case the price has to be included in the JSON with the request, as well as the id of the company to which the booking is allocated, e.g.
    "price": 23.50
    "managingCompanyId": 308
    customer yes The customer object is required. If the id matches an existing customer linked to the same details, then the booking will be linked to the existing customer profile; otherwise a new customer profile will be created. The customer needs to have at least: (i) firstName, (ii) phoneNumberCountryCode, (iii) phoneNumber.
    isAsap yes This field is used to tell the supplier to send a car as soon as they can, regardless of actual booking time. I.e. if the booking is for 20 minutes from now (which could be the contractual notice time agreed), if the supplier has a car available that can be sent immediately, then if isAsap=true the supplier knows that they have to send the car without waiting.
    flightNumber see comments Required for all pickups at airport terminals.

    Note: the ‘via point’ feature is only partially implemented. If you have via points in your booking we suggest that you put the via points addresses in the comments (in human readable form and without coordinates).

    GET /coverage/forLocation/

    Query parameters

    Name Type Description
    lat float Required. Latitude of the location for which you need to know if online bookings are available.
    lng float Required. Longitude of the location for which you need to know if online bookings are available.
    notice float Notice time, in minutes (i.e. “are there any cars available for a pickup within 30 minutes?”).
    toLat float Latitude of the destination of a journey, to know if a journey is covered. In some case only pre-defined routes are allowed from certain locations. Hence even if the coverage on the pickup only had returned that online bookings are available, in some cases it’s useful to re-test this after the destination has been entered so that the user can be redirected to the correct UI before actually attempting to get a price and place a booking.
    toLng float (see above)
    paymentTypeId string See the list of possible values on the Appendices
    fareTypeId string See the list of possible values on the Appendices
    vehicleCategoryId integer Id of the specific vehicle category for which we want to know if online bookings are available.
    forPreBooking boolean Default value=true
    forHailing boolean Default value=false

    Response details

    Field Comments
    coverageFound true: online booking are expected to be available. The user flow can continue.
    false: the location is not covered and a user message should be displayed.
    hailingEnabled true: there are drivers currently online and available to accept bookings.false: bookings sent directly to drivers are not possible as there are no drivers online at the moment. Note: bookings could be possible via the cab offices, with potentially only a few additional minutes for the ETA to pickup.
    preBookingEnabled true: advance bookings (via the cab office who will then dispatch the jobs to the drivers) are possible.
    defaultMinutesForEta ???.
    fareTypeHailing (for hailing) Array with the list of fare allowed fare types for a hailing request. When making the pricing request, use the first fare type with the field isDefault=true (or simply the first fare type if there are no default fare types).
    Use the requireDestination property to know if you need to ask the destination to the user in the UI or whether the destination is optional and you can proceed without it.
    vehicleCategoryHailing (for hailing) Array with the list of vehicle categories available for hailing in the selected location.
    vehicleOnRoad (for hailing) Array with the list of drivers currently online and close to the selected location. You can use this information to display the position of the current drivers or (preferred option) call the /tracking/liveDrivers/xxx service (to be documented).
    fareType (for pre-bookings) Array with the list of fare allowed fare types for a pre-booking request. When making the pricing request, use the first fare type with the field isDefault=true (or simply the first fare type if there are no default fare types).
    vehicleCategory (for pre-bookings) Array with the list of vehicle categories available for pre-booking in the selected location.
    displayBookingCentre When coverageFound=false then some apps offer the option to call an operator to assist the customer. This field tell the user app if the ‘call center operator’ option is available. If yes, an object bookingCentre will appear in the response and will provide the call center information.
